# Timeweb App Platform (Node.js)

Корень Git-репозитория может быть шире; в панели укажи **подпапку с мини-приложением**.

## Project directory

```text
telegram-mini-app
```

## Build command

```bash
npm ci && npm run build && npm ci --omit=dev --prefix server
```

Для сборки фронта задай в окружении сборки (или общих переменных, если Timeweb передаёт их в build):

- `VITE_TELEGRAM_BOT` — например `https://t.me/YourBot`
- `VITE_API_URL` — опционально; если не задано, API с того же домена, что и мини-пп

## Start command

Рабочий каталог процесса — как **project directory** (`telegram-mini-app`).

```bash
npm start --prefix server
```

Node стартует из `server/`, статика читается из `../dist`.

## Переменные окружения (runtime)

| Переменная | Нужна | Описание |
|------------|-------|----------|
| `BOT_TOKEN` | Да | Токен @BotFather |
| `WEB_APP_URL` | Да | HTTPS URL мини-приложения в Telegram |
| `ADMIN_TELEGRAM_IDS` | Да* | ID через запятую для `/grant` и `/revoke` |
| `PORT` | Нет | Обычно задаёт платформа; иначе **3000** |
| `HOST` | Нет | По умолчанию **0.0.0.0** |
| `CORS_ORIGIN` | Нет | Origins через запятую; пусто = любые |

Проверка: `GET /health` → **200** и `{"ok":true}`.

Премиум в `server/data/*.json` на PaaS часто **не переживает** деплой — позже лучше вынести в БД.

## Если приложение не создаётся или деплой красный

Локально у тебя уже проверено: **сборка и `/health` работают** — значит на Timeweb нужно понять **на каком шаге** падает панель.

### Шаг 1 — обязательно: лог деплоя

В панели приложения открой **логи деплоя / сборки** (или историю деплоев) и найди **первую** строку с `error`, `failed`, `npm ERR!`, `EACCES`, `Killed`, `not found`.

Скопируй **30–50 последних строк** и отправь в поддержку или в чат — без этого дальше только угадывание.

Официально Timeweb пишет то же: смотреть лог деплоя в первую очередь — [деплой Express в App Platform](https://timeweb.cloud/docs/apps/deploying-backend-applications/express).

### Шаг 2 — по стадии в логе

| Что в логе | Куда копать |
|------------|-------------|
| `git`, `clone`, `403`, `not found` | Доступ к репозиторию (приватный GitHub, права интеграции). |
| `npm ci` / `package-lock` | Временно замени сборку на: `npm install && npm run build && npm install --omit=dev --prefix server`, закоммить актуальные `package-lock.json` и запушь. |
| `vite`, `tsc`, `build` | Версия Node (поставь **20**), нехватка RAM на тарифе (**Killed** / **ENOMEM**) — тариф с большим RAM. |
| Сборка ОК, падает **старт** | Команда запуска: `npm start --prefix server`, путь к проекту **`telegram-mini-app`** без лишнего `/` в начале. |
| Контейнер перезапускается | Health check **`/health`**, приложение слушает **0.0.0.0** и **PORT** с платформы (у нас в коде так и задумано). |

### Шаг 3 — настройки формы (ещё раз)

- **Путь к проекту:** `telegram-mini-app` (часто без `/` в начале).
- **Node:** **20** (в доке Timeweb для Express указаны 20 / 22 / 24; на 20 меньше сюрпризов).
- **Переменная админа:** точное имя **`ADMIN_TELEGRAM_IDS`** (с `S` на конце).

### Про PM2 на Timeweb

В [документации Timeweb](https://timeweb.cloud/docs/apps/deploying-backend-applications/express) для Express по умолчанию упоминается **PM2** внутри платформы. Тебе **не нужно** настраивать PM2 самому: в поле **«Команда запуска»** должна остаться **твоя** команда `npm start --prefix server`. Если в логе видно, что платформа пытается стартовать **`index.js` из корня** проекта вместо `server` — значит кастомная команда не подхватилась; пересоздай приложение или проверь, что поле «Команда запуска» сохранено.
